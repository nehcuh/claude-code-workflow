# Modern CLI Tools Detection

VibeSOP can automatically detect modern CLI tools installed in your environment and generate `TOOLS.md` documentation to help AI assistants use them effectively.

## Supported Tools

| Modern Tool | Replaces | Purpose |
|-------------|----------|---------|
| [bat](https://github.com/sharkdp/bat) | `cat` | Syntax highlighting, line numbers, git integration |
| [fd](https://github.com/sharkdp/fd) | `find` | Faster, simpler syntax, respects `.gitignore` |
| [rg (ripgrep)](https://github.com/BurntSushi/ripgrep) | `grep` | Fast search that respects `.gitignore` |
| [eza](https://github.com/eza-community/eza) | `ls` | Icons, git status, tree view |
| [dust](https://github.com/bootandy/dust) | `du` | Visual disk usage with intuitive output |
| [duf](https://github.com/muesli/duf) | `df` | Better disk usage visualization |
| [procs](https://github.com/dalance/procs) | `ps` | Process listing with colors and tree view |
| [btop](https://github.com/aristocratos/btop) | `top` | Interactive system monitor with better UI |

## How It Works

### During `vibe init`

When you run `vibe init --platform claude-code` (or `opencode`), VibeSOP will:

1. Scan your PATH for the 8 modern tools
2. Display which tools are found (✅) and which are missing (❌)
3. Ask if you want to generate TOOLS.md

```
🔍 Detecting modern CLI tools...
   Checking bat... ✅ found at /opt/homebrew/bin/bat
   Checking fd... ✅ found at /opt/homebrew/bin/fd
   Checking rg... ✅ found at /opt/homebrew/bin/rg
   Checking eza... ❌ not found
   ...

📊 Found 5 of 8 modern CLI tools

📝 Generate tool recommendations for AI?
   This will create TOOLS.md and help AI use modern tools automatically.
[Y/n]:
```

If you choose **Y**, `TOOLS.md` will be created in your global configuration directory.

### Generated TOOLS.md

The generated file looks like:

```markdown
# Available CLI Tools

Your environment has the following modern tools:

## File Operations

- ✅ `bat` (replaces `cat`)
  - Syntax highlighting, line numbers. Use --paging=never for non-interactive output
  - Use for: Reading code files, Viewing logs
  - Path: `/opt/homebrew/bin/bat`

- ✅ `fd` (replaces `find`)
  - Faster, simpler syntax. Respects .gitignore by default
  - Use for: Finding files by name/pattern, Recursive file search
  - Path: `/opt/homebrew/bin/fd`

## Fallback Strategy

If a modern tool fails with "command not found":
1. Fall back to the traditional tool
2. Inform the user the tool list may be outdated
3. Suggest running `vibe doctor` to refresh

---

Generated by: vibe doctor
Last updated: 2026-03-23 14:30:00
```

This file is automatically referenced in your `CLAUDE.md` / `AGENTS.md` entrypoint.

## Managing Tool Detection

### Check Current Status

```bash
vibe tools status
```

Output:
```
🔧 Modern CLI Tools Status
========================================
  ✅ bat        (cat)
  ✅ fd         (find)
  ✅ rg         (grep)
  ❌ eza        (ls)
  ...

  5/8 tools available
```

### Enable/Disable TOOLS.md

If you opted out during init but want to enable it later:

```bash
vibe tools enable
```

To disable and remove TOOLS.md:

```bash
vibe tools disable
```

### Refresh After Installing New Tools

After installing a new tool (e.g., `brew install eza`):

```bash
vibe doctor
```

This will:
1. Re-detect all tools
2. Update TOOLS.md with the latest information

## Installing Modern Tools

### macOS (Homebrew)

```bash
brew install bat fd ripgrep eza dust duf procs btop
```

### Linux

```bash
# Ubuntu/Debian
sudo apt install bat fd-find ripgrep

# Or use cargo for Rust-based tools
cargo install bat fd-find ripgrep eza du-dust duf procs btop
```

### Alternative Binary Names

Some distributions use different names:

- `fd` may be `fdfind` on Debian/Ubuntu
- `bat` may be `batcat` on some systems

VibeSOP automatically checks for these alternatives.

## Troubleshooting

### Tool not detected but it's installed

1. Verify the binary is in your PATH:
   ```bash
   which bat
   ```

2. If using an alternative name, create a symlink:
   ```bash
   ln -s $(which batcat) ~/.local/bin/bat
   ```

3. Refresh detection:
   ```bash
   vibe doctor
   ```

### TOOLS.md not updating

Check if the file exists:
```bash
ls ~/.claude/TOOLS.md  # or ~/.config/opencode/TOOLS.md
```

If missing, regenerate it:
```bash
vibe tools enable
```

### Permission issues on Windows

On Windows, some tools may require:
- Developer Mode enabled (for symlinks)
- Administrator privileges (for certain installations)

## Technical Details

### Detection Method

VibeSOP uses Ruby native PATH scanning (no subprocess calls):
- Iterates through `ENV['PATH']` directories
- Checks `File.executable?` for each binary
- Supports Windows PATHEXT extensions (`.exe`, `.bat`, `.cmd`)

This approach is:
- **Cross-platform**: Works on macOS, Linux, Windows
- **Fast**: No fork/exec overhead
- **Reliable**: No dependency on `which`/`where` commands

### Alternative Support

Tools can specify alternatives in `core/integrations/modern-cli.yaml`:

```yaml
detection:
  binary: fd
  alternatives: [fdfind]
```

If `fd` is not found, VibeSOP will check for `fdfind`.

### No Caching

Detection runs fresh each time because:
- Tool installation is infrequent
- Fresh detection ensures accuracy
- `vibe doctor` provides easy refresh

## FAQ

**Q: Why doesn't VibeSOP install these tools for me?**
A: Tool installation varies by OS and package manager. We detect what's already installed rather than making assumptions about your system.

**Q: Can I add custom tools?**
A: Edit `core/integrations/modern-cli.yaml` to add new tool definitions. Follow the existing schema.

**Q: Does this work for project-level configuration?**
A: Currently, modern CLI tool detection is global-only (v1.0). Tools are system-level installations, so per-project detection provides limited value.

**Q: What if AI tries to use a tool that's not installed?**
A: The TOOLS.md includes a fallback strategy section. AI should fall back to traditional tools and suggest running `vibe doctor`.
